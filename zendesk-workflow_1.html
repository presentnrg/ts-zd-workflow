TRIGGER ‚Äî FIRES ON GROUP CHANGE

When a ticket is assigned to a group with no specific agent, all group members are notified ‚Äî except the Tech Team group, which has its own trigger.

TRIGGER ‚Äî FIRES ON GROUP CHANGE OR UPDATE

Special notification for the Tech Team group (light agents) with instructions about their limited permissions ‚Äî they can only add internal notes, not public replies.

‚ö°

TRIGGER ‚Äî FIRES ON ANY PUBLIC COMMENT

Notify Requester & CCs of Comment Update

Whenever any public comment is added, the requester and all CCs receive an email with the latest comment and a CSAT rating link. Highest-traffic trigger.

‚ö°

TRIGGER ‚Äî FIRES ON ANY COMMENT

Notify Assignee of Comment Update

When anyone other than the assignee adds a comment (public or internal), the assignee is notified with full context. Ensures agents don't miss customer replies.

‚ö°

TRIGGER ‚Äî FIRES ON SIDE CONVERSATION

Side Conversation Replies ‚Üí Set Ticket to Open

When a side conversation is created, replied to, or reopened, the parent ticket is set back to Open and a specific agent is notified. Ensures side convos don't get lost.

"Solved ‚Äî In Client Admin Hands Now"

When ticket is set to the custom status "Solved - in client admin hands now", it auto-tags with refer_to_system_admin. This routes to a special solve notification path later.
